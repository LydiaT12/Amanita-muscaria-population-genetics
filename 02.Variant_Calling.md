# Reference genome

Variant calling is performed with respect to a reference genome. I use the masked version of the new _Amanita muscaria_ reference genome with the mitogenome included. 

The genome needs to be indexed before reads can be mapped to it, and it requires a sequence dictionary for the GATK pre-processing.

```
samtools faidx Amanita_muscaria_genome_2024_masked.fasta
bwa index Amanita_muscaria_genome_2024_masked.fasta
gatk CreateSequenceDictionary -R Amanita_muscaria_genome_2024_masked.fasta -O Amanita_muscaria_genome_2024_masked.dict
```



# GATK variant calling

## Unmapped bam files

GATK requires data in unmapped bam (ubam) file format. This contains metadata about the sample. I add very little matadata information, because I don't want GATK to make assumptions about the structure of the genetic data based on this metadata. As long as the sample name is preserved, any nesessary metadata can be associated with the sample later, when relevent. 

```
#!/bin/bash -e
module load GATK/4.4.0.0-gimkl-2022a

for filename in ../source_files/trimmed_files/trimmed_*_1.fastq.gz
do
        base=$(basename ${filename} _1.fastq.gz)
        gatk FastqToSam -SM Dunedin -F1 ../source_files/trimmed_files/${base}_1.fastq.gz -F2 ../source_files/trimmed_files/${base}_2.fastq.gz -O ${base}.unmapped.bam -PL ILLUMINA --TMP_DIR ${TMPDIR}
done
```

## Mapping

I have a script which goes through each sample within a site and maps the reads to the reference genome.

```
#!/bin/bash -e
#SBATCH --time=13:00:00         # maximum run time hh:mm:ss
#SBATCH --mem=16G              # maximum memory available to GATK
#SBATCH --cpus-per-task=4

module load BWA
module load SAMtools

setname=$1
echo "${setname}"

for filename in ../source_files/trimmed_files/trimmed_${setname}*_1.fastq.gz
do
        base=$(basename ${filename} _1.fastq.gz)
        echo "mapping $base"

        bwa mem -M -t ${SLURM_CPUS_PER_TASK} 1.ref_genome/Amanita_muscaria_genome_2024_masked.fasta ../source_files/trimmed_files/${base}_1.fastq.gz ../source_files/trimmed_files/${base}_2.fastq.gz > 3.mapped_reads/${base}.sam
        samtools view -S -b 3.mapped_reads/${base}.sam > 3.mapped_reads/${base}.bam
done
```

I can submit this as four jobs at once with the following script. The code for site T was run first, seperately, to test that it was working. 

```
for setname in S F H W; do sbatch run_mapping.sl ${setname}; done
```



## Pre-processing steps

The pre-processing steps are (1) mapping to reference (we did the mapping already, but have some filtering steps which come under this heading). (2) marking duplicate reads and (3) base quality score recalibration (BQSR). We skip the BSQR (for now) because we have no data on variable sites. We could bootstrap this later, after an initial round of variant calling, but I don't really think it'll be worth the hassle. https://gatk.broadinstitute.org/hc/en-us/articles/360035535912-Data-pre-processing-for-variant-discovery
After pre-processing, we will do variant calling. The first step of this calls haplotypes in each sample independently. We do this step along with the pre-processing, since the jobs are per-sample and it's the easiest way to batch it. 
https://gatk.broadinstitute.org/hc/en-us/articles/360035535932-Germline-short-variant-discovery-SNPs-Indels

Do a test batch with:
```
for setname in trimmed_T11_DKDN230032144-1A_HLK32DSX7_L1; do sbatch run_preprocessing_steps.sl $setname; done
for setname in 2.ubams/trimmed_*; do sbatch run_preprocessing_steps.sl $setname; done
```

```
#!/bin/bash -e
#SBATCH --time=04:21:00         # maximum run time hh:mm:ss
#SBATCH --mem=16G              # maximum memory available to GATK

module load GATK
module load SAMtools

filename=$1
file=$(basename ${filename} .unmapped.bam)
echo "${file}"

samtools view -b -e '!([XA]||[SA])' 3.mapped_reads/${file}.bam > 3.mapped_reads/${file}.unique.bam
gatk MergeBamAlignment -ALIGNED 3.mapped_reads/${file}.unique.bam -UNMAPPED 2.ubams/${file}.unmapped.bam -O 4.preprocessing/${file}.merged.bam -R 1.ref_genome/Amanita_muscaria_genome_2024_masked.fasta
gatk MarkDuplicatesSpark -I 4.preprocessing/${file}.merged.bam -O 4.preprocessing/${file}.markeddups.bam
gatk HaplotypeCaller -R 1.ref_genome/Amanita_muscaria_genome_2024_masked.fasta -I 4.preprocessing/${file}.markeddups.bam -O 5.hapcalling/${file}.g.vcf.gz  -ERC GVCF
```

## Combine gvcf files

This is the step where we combine all of the files generated per-individual and treat them as a joint sample from a population.

First, I make a document which contains details of where each file exists and what sample it relates to. This is a text file with sample_name--tab--path_to_sample_vcf per line. This is called samplenames.txt, and the first five lines are

```
F2      5.hapcalling/trimmed_F2_DKDN230032153-1A_HLK32DSX7_L1.g.vcf.gz
F4      5.hapcalling/trimmed_F4_DKDN230032155-1A_HLK32DSX7_L1.g.vcf.gz
F5      5.hapcalling/trimmed_F5_DKDN230032156-1A_HLK32DSX7_L1.g.vcf.gz
F6      5.hapcalling/trimmed_F6_DKDN230032157-1A_HLK32DSX7_L1.g.vcf.gz
F7      5.hapcalling/trimmed_F7_DKDN230032158-1A_HLK32DSX7_L2.g.vcf.gz
```

To run GenomicsDBImport I need an input for -L, the genomic regions over which to do the calling. I can pass a file to this arguement. First, I need to create a file with the appropriate arguments. I want the calling to occur over whole contigs, so I need a list of the contigs. If I use a list of all the scaffolds, I get an error message that calling is less effective with more than 100 named genomic regions. Since the reference genome has an L50 of 28, the longest 28 scaffolds cover half the genome. I start by calling variants over just these 28 contigs, knowing that this comprised half the genome. The code below generates a file with the names of the first 28 scaffolds.

```
for i in {1..28}; do echo "Scaffold${i}_Amanita_muscaria"; done > scaffoldnames.part1.list
```

Now I run GenomicsDBImport. The database name cannot already exist, and partial calling creates errors later on. 

```
gatk GenomicsDBImport --genomicsdb-workspace-path gatk_genomicsDB_1to28   --batch-size 20 --sample-name-map samplenames.txt -L scaffoldnames.part1.list
```

## Joint calling

Now that I have a database, I can call the joint variants. 

```
gatk GenotypeGVCFs -R 1.ref_genome/Amanita_muscaria_genome_2024_masked.fasta -V gendb://gatk_genomicsDB_1to28  -O 6.variant_files/partialdataset_1to28.vcf.gz   --tmp-dir temp_for_GATK/
```
