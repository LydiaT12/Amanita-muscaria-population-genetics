# Filtering

I want to filter out anything that looks like it might be an error. To do this, I need to identify outliers. The first things to check are read depth and mapping quality. These give us an idea of how much we trust the output from the haplotype calling, and may suggest areas in the reference genome assembly which are messy. 

## Indels

Indels are more likely than SNPs to be errors [where did I read that?]. They comprise a small proportion of the variants. I play it safe, and filter them all out. 

```
vcftools --gzvcf 1.subset_files/test_scaffold1.vcf.gz --max-alleles 2 --recode -c --recode-INFO-all | gzip -c > 1.subset_files/scaffold1_biallelic.vcf.gz
vcftools --gzvcf 1.subset_files/test_scaffold1.vcf.gz --remove-indels --recode -c --recode-INFO-all | gzip -c > 1.subset_files/scaffold1_noindels.vcf.gz
vcftools --gzvcf 1.subset_files/scaffold1_biallelic.vcf.gz --remove-indels --recode -c --recode-INFO-all | gzip -c > 1.subset_files/scaffold1_biallelic_noindels.vcf.gz
```

The -recode-INFO-all tag is needed to keep important metadata when changing VCF files in vcftools.

## SNP quality

The likelihood that the variant calling algorithm puts that the called SNP is correct is measured by a mapping quality score. I want to filter out all variants with a low mapping quality. 

Read depth measures the number of reads which aligned to a site. It should be fairly even across the genome. Regions where the read depth differs from the baseline value are more likely to have erroneous SNPs - due to misassembly of the reference genome or too few reads to correctly infer heterozygosity when it is present, or excess reads inaccurately mapped to the site and introducing spurious "variation". I need to find out what the baseline read depth is for these mappings and filter out everything too far from the baseline. 

The first step is to find out what the distribution of read depths and mapping qualities is, so I can choose thresholds for filtering. 

## Thinning

Thin the variants so that the vcf is small enough to work with in R, while still covering all 28 super-contigs. 

```
vcftools --gzvcf scaffolds_1to28_filtered.vcf.gz --thin 500  --recode -c --recode-INFO-all | gzip -c > 1.subset_files/thinned_1to28.vcf.gz
# Ensure that no two variants are less than 500 bases apart
```




# Convert to single-scaffold datasets

For some analyses I will want datasets which comprise a single scaffold. 

```
vcftools --gzvcf scaffolds_1to28_filtered.vcf.gz --chr Scaffold1_Amanita_muscaria  --recode -c --recode-INFO-all | gzip -c > 1.subset_files/test_scaffold1.vcf.gz
```
The -recode-INFO-all tag is needed to keep important metadata

