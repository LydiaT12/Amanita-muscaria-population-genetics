The sequence data are paired-end, meaning that we have forward and reverse sequences for each fragment. The R1 file contains sequences in one direction and the R2 file in the other direction.

The data are in fastq files. The .gz is a zip format. They can be unzipped using ```gunzip $filename```. The fastq format consists of four lines:
1. '@' and metadata
2. The DNA sequence
3. '+'
4. Quality scores for each base in the sequence

   
## CheckSum

The files ```checkSize.xls``` and ```MD5.txt``` contain the checksum values and ```MD5.zip``` contains a tool for checking the checksums. The Readme has the following information:

> An MD5 checksum is a 32-character hexadecimal number that is computed on a file. If two files have the same MD5 checksum value, it is highly probable that the two files are the same. It is generally used to check data integrity.
> In Linux or Mac system, the following command could be used directly to check MD5(If md5sum command is not available, please follow the manual inside the MD5.zip package to install.):
    $ md5sum -c MD5.txt    (it may take a while)


I ran

```
md5sum -c MD5.txt
```

and all the files got an output of ```OK```

## Adapter trimming
<!--cutadapt/4.4-gimkl-2022a-Python-3.11.3-->
We use the module cutadapt to trim off adapters and removing reads shorter than 50bp. We need to give it the actual adapter sequences to remove

For paired-end reads: cutadapt -a ADAPT1 -A ADAPT2 [options] -o out1.fastq -p out2.fastq in1.fastq in2.fastq
(.gz compressed file type supported and automatically detected)

The Novogene documentation tells us the adaptor sequences are
  5' Adapter:
  5'-AGATCGGAAGAGCGTCGTGTAGGGAAAGAGTGTAGATCTCGGTGGTCGCCGTATCATT-3'
  3' Adapter:
  5'-GATCGGAAGAGCACACGTCTGAACTCCAGTCACGGATGACTATCTCGTATGCCGTCTTCTGCTTG-3'
  
First test things on a single file, then when that works do all files.

```
#!/bin/bash -e

#SBATCH --job-name=SlurmTest # job name (shows up in the queue)
#SBATCH --time=03:00:00      # Walltime (HH:MM:SS)
#SBATCH --mem=1G          # Memory in GB

module load cutadapt
for filename in *1.fq.gz
do
   file=$(basename ${filename} 1.fq.gz)

   cutadapt -a AGATCGGAAGAGCGTCGTGTAGGGAAAGAGTGTAGATCTCGGTGGTCGCCGTATCATT -A GATCGGAAGAGCACACGTCTGAACTCCAGTCACGGATGACTATCTCGTATGCCGTCTTCTGCTTG  -q 25 -o trimmed_${file}1.fastq.gz --minimum-length 50:50   -p  trimmed_${file}2.fastq.gz  ${file}1.fq.gz  ${file}2.fq.gz

done
```
